## Commit Message Convention

### Format
```
<Type>(<Scope>): <Summary>

<Body - optional>

<Footer - optional>
```

### Types
- `feat:` — New feature
- `fix:` — Bug fix
- `refactor:` — Code reorganization (no behavior change)
- `style:` — Formatting, missing semicolons
- `test:` — New tests or test fixes
- `docs:` — Documentation changes
- `chore:` — Build tools, dependencies

### Scope
- `agent` — Agent-related changes
- `api` — REST API endpoints
- `db` — Database/Prisma changes
- `ui` — Frontend components
- `broker` — Broker API integration
- `backtest` — Backtesting engine

### Examples
```
feat(agent): add Elliott Wave master with Fibonacci analysis

fix(db): resolve Prisma type generation for new models

refactor(orchestrator): simplify vote aggregation logic

docs(readme): update setup instructions for Day 3
```

## After First Push

### GitHub Settings to Configure
1. **Branch Protection**
   - Settings → Branches → Add rule for `main`
   - Require PR reviews before merging
   - Require status checks to pass

2. **CI/CD Setup**
   - Create `.github/workflows/test.yml` for automated tests
   - Create `.github/workflows/deploy.yml` for auto-deployment

3. **Secrets Management**
   - Settings → Secrets → Add SUPABASE credentials

4. **Release Management**
   - Create releases for Day 1, Day 2, Day 3
   - Tag with v1.0.0, v1.1.0, v1.2.0

## Access & Authentication

### GitHub CLI Setup
```bash
# Install GitHub CLI
# https://cli.github.com/

# Login
gh auth login

# Verify
gh auth status
```

### HTTPS vs SSH
- **HTTPS:** Use personal access token (no SSH setup needed)
- **SSH:** More secure, requires key pair setup

```bash
# For HTTPS with credentials stored
git config --global credential.helper store

# For SSH (recommended)
ssh-keygen -t ed25519 -C "your.email@gmail.com"
# Add public key to GitHub Settings → SSH Keys
```

## Emergency Git Commands

### If push fails
```bash
# Check status
git status

# Pull latest
git pull origin main

# Force push (use carefully!)
git push -u origin main --force-with-lease
```

### If merge conflict
```bash
# Abort merge
git merge --abort

# Manual resolution
# Edit files, then:
git add .
git commit -m "fix: resolve merge conflict"
git push origin main
```

### If committed something accidentally
```bash
# Undo last commit (keep changes)
git reset --soft HEAD~1

# Or undo and discard changes
git reset --hard HEAD~1

# After fix:
git commit -m "fix: corrected previous commit"
git push origin main
```
